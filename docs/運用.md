# 運用手順書

## 前提条件

- Docker + Docker Compose
- Python 3.10+ (テスト実行のみローカル)

---

## 1. 初期データ投入 (seed)

sklearn の California Housing データセットを SQLite に格納する。

```bash
make seed
```

- 出力先: `data/dev.db`
- テーブル: `training_data` (16,512行), `test_data` (4,128行)
- 分割: 8:2 (random_state=42 で固定)

> 初回のみ実行。再実行すると既存テーブルを上書き (replace) する。

---

## 2. モデル学習

```bash
make train
```

### 出力

各実行に一意な Run ID (`YYYYMMDD_HHMMSS_{UUID}`) が付与される。

```
models/
├── {run_id}/
│   ├── model.lgb           # LightGBM モデル
│   └── metrics.json        # 評価指標 (RMSE, R², run_id)
└── latest -> {run_id}      # 最新 run へのシンボリックリンク
```

学習を複数回実行するとモデルが蓄積され、比較が可能。
精度 (RMSE, R²) は SQLite の `model_runs` テーブルにも記録される。

### 前処理

学習前にデータに対して以下の前処理が自動適用される (`ml/pipeline/preprocess.py`):

1. **欠損値補完** — 中央値で補完
2. **外れ値キャップ** — Population, AveOccup, AveRooms, AveBedrms を 99パーセンタイルでカット
3. **対数変換** — Population, AveOccup に `log1p()` を適用

### 特徴量エンジニアリング

前処理の後に、既存特徴量から新しい特徴量を自動生成する (`ml/pipeline/feature_engineering.py`):

| 生成特徴量 | 計算式 | 意味 |
|---|---|---|
| BedroomRatio | AveBedrms / AveRooms | 寝室が占める割合 |
| RoomsPerPerson | AveRooms / AveOccup | 1人あたりの部屋数 |

推論 API でも同じ特徴量生成 + 対数変換が自動適用される。

### エラーハンドリング

`ml/pipeline/main.py` で各ステップが保護されている:
- データ取得失敗 → エラーログ + 終了
- W&B 初期化失敗 → 警告ログで継続（学習は止めない）
- 学習失敗 → エラーログ + 終了
- W&B ログ失敗 → 警告ログで継続

### W&B ログ

- `.env` に `WANDB_API_KEY` を設定するとクラウドに送信
- 未設定時は offline モードで `src/ml/wandb/` 配下にローカル保存
- W&B なしでも精度評価・モデル保存は正常に動作する

```bash
# 後からクラウド同期する場合
wandb sync src/ml/wandb/offline-run-XXXXXXXX
```

### LightGBM パラメータ

`src/ml/trainer/train.py` の `LGB_PARAMS` で管理:

| パラメータ | 値 |
|---|---|
| objective | regression |
| metric | rmse |
| learning_rate | 0.05 |
| num_leaves | 31 |
| num_boost_round | 300 |

## 3. 推論 API 起動

```bash
make serve
```

### エンドポイント

| Method | Path | 説明 |
|---|---|---|
| GET | `/` | 予測フォーム (Web UI) |
| GET | `/health` | ヘルスチェック |
| POST | `/predict` | 住宅価格予測 (JSON API) |

### Web UI

ブラウザで http://localhost:8000/ にアクセスすると予測フォームが表示される。
特徴量を入力して Predict ボタンを押すと予測結果が表示される。

### API リクエスト例

```bash
curl -X POST http://localhost:8000/predict \
  -H "Content-Type: application/json" \
  -d '{
    "MedInc": 8.3,
    "HouseAge": 41,
    "AveRooms": 6.9,
    "AveBedrms": 1.0,
    "Population": 322,
    "AveOccup": 2.5,
    "Latitude": 37.88,
    "Longitude": -122.23
  }'
```

### レスポンス例

```json
{"predicted_price": 4.2103}
```

> 価格の単位は $100,000。4.2103 = 約 $421,030。

## 5. Docker 管理

```bash
make build          # イメージビルド
make down           # 全サービス停止
make clean          # 停止 + 生成ファイル削除 (data/dev.db, models/, wandb/)
make all            # build → seed → train
```

### Docker サービス

| サービス | Dockerfile | ポート | 用途 |
|---|---|---|---|
| seed | Dockerfile.trainer | — | SQLite データ投入 (run して終了) |
| trainer | Dockerfile.trainer | — | 学習 (seed 完了後に実行) |
| api | Dockerfile.api | 8000 | FastAPI 推論 + Web UI |

## 7. 環境変数一覧

| 変数 | 用途 | デフォルト |
|---|---|---|
| `DATA_SOURCE` | データソース種別 | `sqlite` |
| `SQLITE_PATH` | SQLite ファイルパス | `data/dev.db` |
| `WANDB_API_KEY` | W&B API キー | (空 → offline) |
| `WANDB_PROJECT` | W&B プロジェクト名 | `california-housing` |
| `MODEL_DIR` | モデル出力先 | `models` |
| `MODEL_PATH` | API が読むモデルパス | `models/latest/model.lgb` |
