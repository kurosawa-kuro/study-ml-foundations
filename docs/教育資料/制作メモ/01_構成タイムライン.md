## 章ごとの狙いと構成詳細

### Ch 0 — オープニング（0:00–0:30）

- タイトルロゴ + BGM（3〜5 秒）
- 話者自己紹介 or プロジェクト概要（10 秒）
- 「この動画のゴール」テロップ 3 点提示
  1. 機械学習の **基本用語** が分かる
  2. **パイプライン全体** の流れが掴める
  3. 実際に **動かして試せる**

### Ch 1 — なぜ機械学習か（0:30–1:30）

- スライド 01: 従来プログラミング vs ML の比較表
- スライド 00-C: 今日の見取り図
- 「ルールを書く → データから学ぶ」のパラダイム転換
- 本プロジェクトの題材（California Housing）を 1 行で紹介
- **強調**: ここで全体の段階を先に見せる

### Ch 2 — モデルと特徴量（1:30–4:00）

- スライド 02: モデルとは（入力 → 関数 → 出力）
- スライド 03: 特徴量一覧表（8 項目）
- スライド 04: 特徴量エンジニアリング（BedroomRatio / RoomsPerPerson）
- **強調**: 「ドメイン知識を数式にする工程」

### Ch 3 — データ分割と前処理（4:00–6:00）

- スライド 04-B: 「入力設計」から「データ準備」への切り替えを明示
- スライド 05: 訓練 80% / テスト 20%
- スライド 06: 前処理 3 ステップ（欠損値補完・外れ値キャップ・対数変換）
- コードちら見: `preprocess.py` の関数シグネチャを 5 秒ほど表示
- **強調**: 特徴量設計と前処理は別の責務であること

### Ch 4 — 学習アルゴリズム（6:00–8:00）

- スライド 06-C: 「データ準備」から「モデル構築」への切り替えを明示
- スライド 07: アルゴリズム分類表
- LightGBM の位置付け（木ベース）
- スライド 08: ハイパーパラメータ 3 つ
- スライド 09: 学習コード 3 行で「何が起きるか」
- **強調**: ここからは入力ではなく予測器そのものを作る段階

### Ch 5 — 評価指標（8:00–9:30）

- スライド 09-B: 「構築」から「品質確認」への切り替えを明示
- スライド 10: RMSE / R² の意味
- スライド 11: オフライン / オンライン評価
- **強調**: 「numpy で自前実装」→ ライブラリに頼らず中身を理解できる学習プロジェクトであること

### Ch 6 — 推論とバージョニング（9:30–11:00）

- スライド 12: 推論とは（学習は重い・推論は軽い）
- スライド 12-B: 「品質確認」から「利用・運用」への切り替えを明示
- スライド 13: Run ID とディレクトリ構造
- `models/latest` シンボリックリンク の仕組み

### Ch 7 — 実験管理 W&B（11:00–12:30）

- スライド 14: W&B の役割（計算ではなく**蓄積と可視化**）
- offline モードで API キーなしでも動く
- 画面例（可能なら実際のダッシュボードスクショ）

### Ch 8 — デモ（12:30–17:00）★本動画のハイライト

1. `make seed` — SQLite に投入（30 秒）
2. `make train` — 学習ログを読み解く（60 秒）
3. `models/latest` ディレクトリ確認（20 秒）
4. `make serve` — API 起動（30 秒）
5. ブラウザで **予測フォーム** を使う（40 秒）
6. **/metrics ページ** で RMSE/R² 確認（30 秒）
7. **/data ページ** で学習データサンプル確認（30 秒）

### Ch 9 — まとめ（17:00–18:30）

- スライド 15: 概念対応表（ML ↔ ソフトウェア開発）
- スライド 16: 技術スタック一覧
- 「次のステップ」提案（Optuna / クラウド連携 / MLOps）

### E — エンディング（18:30–19:00）

- リポジトリ URL テロップ
- チャンネル登録 / 次回予告
- アウトロ BGM

## 撮影・編集メモ

- **画面収録解像度**: 1920×1080（コード表示のため）
- **ターミナルフォント**: 14pt 以上推奨
- **ブラウザ**: ダーク UI に合わせて背景色を統一
- **テロップ色**: ライム (#a3e635) をアクセントに使用（UI と統一感）
- **BGM**: 控えめなローファイ or アンビエント
- **カット指示**: スライド切替時 0.2 秒フェード、デモ中は即カット
