# コードデモ・シナリオ

**章:** Ch 8（12:30〜17:30、約 5 分）
**画面:** ターミナル + ブラウザ（並列レイアウト or 切替）
**前提:** `make build` は事前完了、キャッシュ warm 状態

---

## 全体フロー

```
0. アプリ概要    →  何を動かすのかを最初に提示
1. make seed    →  SQLite 投入を確認
2. make train   →  学習ログを読み解く
3. models/      →  Run ID ディレクトリ構造
4. make serve   →  API 起動ログを確認
5. ブラウザ     →  /      (予測)
                   /metrics (精度)
                   /data    (学習データ)
```

---

## Step 0: アプリケーション概要（30 秒）

### 画面

- アーキテクチャ図 1 枚（全画面）
- 続けて、完成後の Web UI スクリーンショット 3 枚（`/` / `/metrics` / `/data`）を 2 秒ずつ連続表示

### ナレーション合流

> これから動かすのは、カリフォルニア住宅価格を予測する機械学習パイプラインと、その予測 API です。
> 構成はシンプルで、データは SQLite、学習は LightGBM、推論 API は FastAPI、全体を Docker Compose でまとめています。
> 画面は 3 つ。トップページが予測フォーム、`/metrics` が精度表示、`/data` が学習データビューアです。
> 具体的な動かし方は、この後 `make` コマンドを 3 つ打つだけ、という流れで見ていきます。

### アーキテクチャ図に含める要素

```
SQLite (data/dev.db)
   ↓ seed
ml/pipeline → ml/trainer → ml/evaluation
   ↓
models/{run_id}/model.lgb
   ↓ lifespan load
FastAPI (port 8000)  ←  ブラウザ
```

📺 テロップ: `SQLite × LightGBM × FastAPI × Docker Compose`

## Step 1: make seed（30 秒）

### 操作

```bash
make clean
make seed
```

### ナレーション合流

> まずはデータの投入です。`make seed` を実行すると、scikit-learn のカリフォルニア住宅データセットを SQLite に書き込みます。

### 画面操作

- ターミナルフルスクリーン
- 出力の `Saving ... rows to training_data` 行にズーム（0.5 秒）
- 完了後に別ターミナルで以下を実行し、件数を確認

```bash
sqlite3 data/dev.db "SELECT COUNT(*) FROM training_data; SELECT COUNT(*) FROM test_data;"
```

📺 テロップ: `訓練 16,512 件 / テスト 4,128 件`

---

## Step 2: make train（60 秒）

### 操作

```bash
make train
```

### ナレーション合流

> 次に学習です。`make train` で、前処理・特徴量エンジニアリング・LightGBM 学習・評価・モデル保存、が一気に実行されます。

### 注目させる出力行（順序通り）

| タイミング | ログ行 | テロップ |
|---|---|---|
| 0:05 | `Loading data...` | `Repository がデータ取得` |
| 0:08 | `train: 16512 rows, test: 4128 rows` | `80/20 分割済み` |
| 0:12 | `Preprocessing...` | `前処理開始` |
| 0:14 | `Capped Population: ... rows` | `外れ値キャップ` |
| 0:16 | `Log-transformed Population` | `対数変換` |
| 0:20 | `Feature engineering...` | `8 → 10 特徴量` |
| 0:25 | `[50] valid_0's rmse: ...` | `学習曲線: 50 本目の木` |
| 0:35 | `[300] valid_0's rmse: 0.46...` | `300 本で収束` |
| 0:45 | `Run ID: 20260417_...` | `一意な ID を付与` |
| 0:48 | `Metrics: RMSE=0.46, R²=0.83` | `アール二乗 0.83` |
| 0:52 | `Model saved to .../model.lgb` | `モデル保存完了` |

### 画面操作

- 学習ログ出力中、RMSE の値が徐々に下がっていく様子を強調（RMSE の数値だけをハイライト）
- 最終値にピタッと止まったタイミングで 0.5 秒ポーズ

## Step 3: models/ ディレクトリ確認（20 秒）

### 操作

```bash
tree models/ -L 2
# または
ls -la models/ && ls models/latest/
```

### 期待出力

```
models/
├── 20260417_060339_880f37/
│   ├── metrics.json
│   └── model.lgb
└── latest -> 20260417_060339_880f37
```

### ナレーション合流

> 学習結果は Run ID 付きのディレクトリに保存され、`latest` シンボリックリンクが自動で最新を指します。

### 画面操作

- `latest ->` の矢印に赤マーカー
- `cat models/latest/metrics.json` を実行して中身を表示（2 秒）

📺 テロップ: `metrics.json = このモデルの成績表`

---

## Step 4: make serve（30 秒）

### 操作

```bash
make serve
```

### 注目させる出力行

| ログ行 | テロップ |
|---|---|
| `Building ... api` | `Docker ビルド` |
| `Model loaded from /app/models/latest/model.lgb` | `起動時にモデルロード` |
| `Uvicorn running on http://0.0.0.0:8000` | `API 起動完了` |

### ナレーション合流

> `make serve` で API が起動します。FastAPI の lifespan フックで、起動時にモデルが 1 回だけメモリにロードされるので、リクエストごとの読み込みコストはゼロです。

📺 テロップ: `localhost:8000`

## Step 5: ブラウザで `/` 予測フォーム（40 秒）

### 画面操作

1. ブラウザで `http://localhost:8000/` を開く
2. サイドバー（Predict / Metrics / Data）を指差し（1 秒）
3. デフォルト値のまま「Predict」をクリック
4. 結果カードの `$419,590` にズーム

### ナレーション合流

> トップページは予測フォームです。特徴量 8 つを入力して Predict を押すと、LightGBM が推論して、ドル単位で予測価格を返します。
> デフォルト値は、世帯収入が高くて海辺に近いエリアの設定なので、42 万ドル前後の高めの予測が出ます。

### 追加デモ（任意）

- `MedInc` を `2.0`（低所得）に変えて再実行
- 予測値が大きく下がることを示す

📺 テロップ: `MedInc が大きいほど予測価格も上がる`

---

## Step 6: `/metrics` ページ（30 秒）

### 画面操作

1. サイドバーから Metrics をクリック
2. RMSE / R² のカードにズーム
3. run_id 表示にフォーカス

### ナレーション合流

> Metrics ページでは、最新モデルの精度が見られます。
> RMSE が 0.46、R² が 0.83 と表示されていますが、これは先ほど学習した直後の数値と一致しています。
> `models/latest/metrics.json` を読んで表示しているだけの、シンプルな実装です。

📺 テロップ: `R² 0.83 = 住宅価格のばらつきの 83% を説明できている`

---

## Step 7: `/data` ページ（30 秒）

### 画面操作

1. サイドバーから Data をクリック
2. 先頭 50 行の表をスクロール（上から下へ）
3. `train` / `test` タブを切替
4. `Price` カラム（ライム色）にフォーカス

### ナレーション合流

> Data ページでは、学習に使った実際のデータを覗けます。
> 8 つの特徴量と、目的変数である Price が並んでいます。
> タブで訓練データとテストデータを切り替えられます。
> 「このデータからモデルが学んだのか」が直感的に分かり、教育用途に便利です。

📺 テロップ: `Price = 目的変数（$100k 単位）`

## トラブルシューティング（撮影当日用）

| 症状 | 即時対応 |
|---|---|
| `make train` で pandas エラー | trainer イメージ再ビルド: `docker compose build trainer` |
| API が 8000 で起動しない | 既存プロセス確認: `ss -tlnp \| grep 8000` |
| /predict が 500 エラー | 特徴量数の不一致 → `make build && make train` で再学習 |
| /data が空 | `./data:/app/data` マウント確認（docker-compose.yml） |
| /metrics が 「見つかりません」 | `models/latest/metrics.json` の存在確認 |
