# 図解（Mermaid / PlantUML）

動画内で使う図の定義。Mermaid Live Editor や PlantUML でレンダリング可能。
最終的に PNG/SVG に書き出して編集ソフトに取り込む想定。

---

## Diagram 1: 従来プログラミング vs 機械学習（Ch 1）

```mermaid
flowchart LR
    subgraph Traditional["従来のプログラミング"]
        D1[データ] --> P1[人が書いたルール]
        P1 --> O1[出力]
    end

    subgraph ML["機械学習"]
        D2[データ] --> T[学習]
        O2[期待する出力] --> T
        T --> M[モデル<br/>（学習済みルール）]
        D3[新しいデータ] --> M
        M --> P2[予測出力]
    end

    classDef traditional fill:#1f2937,stroke:#9ca3af,color:#f5f5f5
    classDef ml fill:#1a2e05,stroke:#a3e635,color:#f5f5f5
    class D1,P1,O1 traditional
    class D2,D3,O2,T,M,P2 ml
```

**使い方:** Ch 1 冒頭でスライドに全面表示（5 秒）。左ブロックを先に、右ブロックを 1 秒後に点灯させるとより効果的。

## Diagram 2: 全体パイプライン（Ch 2 冒頭 or Ch 8 デモ導入）

```mermaid
flowchart TB
    Data[("SQLite<br/>data/dev.db")]
    Repo["Repository<br/>(fetch_train/test)"]
    Pre["前処理<br/>欠損値・外れ値・対数"]
    FE["特徴量<br/>エンジニアリング<br/>8→10"]
    Train["LightGBM 学習"]
    Eval["評価<br/>RMSE / R²"]
    Models[("models/<br/>{run_id}/")]
    Latest["models/latest<br/>（シンボリックリンク）"]
    API["FastAPI<br/>/ /predict /metrics /data"]
    WB[("W&B<br/>実験ログ")]

    Data --> Repo --> Pre --> FE --> Train --> Eval
    Train --> Models
    Models --> Latest
    Latest --> API
    Eval --> WB
    Eval --> Models

    classDef storage fill:#0f172a,stroke:#a3e635,color:#f5f5f5
    classDef process fill:#1a2e05,stroke:#a3e635,color:#f5f5f5
    classDef serve fill:#365314,stroke:#a3e635,color:#ffffff
    class Data,Models,Latest,WB storage
    class Repo,Pre,FE,Train,Eval process
    class API serve
```

**使い方:** Ch 8 デモの直前に全体像として再登場させる。各ノードを順番にハイライトしながらナレーションを合わせると効果的。

## Diagram 3: 訓練/テスト分割（Ch 3）

```mermaid
flowchart LR
    All[("全データ<br/>20,640 件")]
    All --> Train[("訓練データ<br/>16,512 件<br/>80%")]
    All --> Test[("テストデータ<br/>4,128 件<br/>20%")]

    Train --> Learn["学習<br/>パラメータ調整"]
    Test --> Score["性能評価<br/>（学習に未使用）"]

    Learn -.-> Model["モデル"]
    Model --> Score

    classDef data fill:#0f172a,stroke:#a3e635,color:#f5f5f5
    classDef proc fill:#1a2e05,stroke:#a3e635,color:#f5f5f5
    class All,Train,Test data
    class Learn,Score,Model proc
```

**使い方:** 円グラフ 80/20 の代わりにこちらを採用しても良い。テストデータが学習に「使われない」ことを矢印で明示。

## Diagram 4: モデルのバージョニング（Ch 6）

```mermaid
flowchart TB
    subgraph Models["models/"]
        R1["20260416_144136_4630da/<br/>├─ model.lgb<br/>└─ metrics.json"]
        R2["20260417_060339_880f37/<br/>├─ model.lgb<br/>└─ metrics.json"]
        L["latest（シンボリックリンク）"]
    end

    R1 -.->|以前の学習| R1
    R2 -.->|最新の学習| R2
    L ==>|現在指している| R2

    API["FastAPI"] --> L

    classDef old fill:#1f2937,stroke:#6b7280,color:#9ca3af
    classDef new fill:#1a2e05,stroke:#a3e635,color:#ffffff
    classDef link fill:#365314,stroke:#a3e635,color:#ffffff
    class R1 old
    class R2 new
    class L link
```

**使い方:** Slide 13 の代替として使用可。`latest` が動的に付け替わる様子を、再学習後にアニメーション化すると良い。

## Diagram 5: W&B の位置付け（Slide 14-B）

```mermaid
flowchart LR
    subgraph Calc["精度の計算（本プロジェクト）"]
        NP["numpy"]
    end

    subgraph Track["蓄積・可視化（クラウド or offline）"]
        WB["W&B<br/>ダッシュボード"]
    end

    Y1["y_true<br/>（テストデータの実測値）"] --> NP
    Y2["y_pred<br/>（モデルの予測値）"] --> NP
    NP --> M["RMSE / R²<br/>（数値）"]
    M --> WB
    WB --> V1["学習曲線"]
    WB --> V2["実験比較"]
    WB --> V3["履歴"]

    classDef input fill:#0f172a,stroke:#9ca3af,color:#f5f5f5
    classDef calc fill:#1a2e05,stroke:#a3e635,color:#ffffff
    classDef wb fill:#365314,stroke:#a3e635,color:#ffffff
    classDef output fill:#1f2937,stroke:#a3e635,color:#f5f5f5
    class Y1,Y2 input
    class NP,M calc
    class WB wb
    class V1,V2,V3 output
```

**使い方:** Slide 14-B に配置。「W&B は計算しない。蓄積するだけ」というメッセージを視覚化。

## Diagram 6: リクエストから推論結果までの流れ（Ch 6 推論）

```mermaid
sequenceDiagram
    actor U as ユーザー
    participant W as Web UI
    participant A as FastAPI
    participant P as preprocess_input
    participant F as engineer_features_input
    participant M as LightGBM モデル

    U->>W: 特徴量 8 個を入力
    W->>A: POST /predict (JSON)
    A->>P: 対数変換など
    P-->>A: 前処理後の値
    A->>F: BedroomRatio / RoomsPerPerson 追加
    F-->>A: 10 特徴量
    A->>M: booster.predict(features)
    M-->>A: 予測値（$100k 単位）
    A-->>W: {"predicted_price": 4.2}
    W->>U: $420,000 を表示
```

**使い方:** Step 5（予測フォームのデモ）の裏で、このシーケンス図をピクチャインピクチャで小さく表示しても良い。

## レンダリング手順

### Mermaid

```bash
# Mermaid CLI をインストール
npm install -g @mermaid-js/mermaid-cli

# 各図を PNG に変換
mmdc -i diagram.mmd -o diagram.png -t dark -b '#0a0a0a'
```

または Mermaid Live Editor（https://mermaid.live）に貼り付けて、Theme: dark、Background: transparent でエクスポート。

### PlantUML（代替）

Mermaid で表現しきれない複雑な図がある場合は PlantUML を併用。今回の範囲では不要。

## スタイル統一

動画内の図は UI と色を揃えると一体感が出る：

- 背景: `#0a0a0a`（UI のカード色）
- アクセント: `#a3e635`（ライム）
- サブテキスト: `#9ca3af`
- プライマリテキスト: `#f5f5f5`

図の書き出し解像度は **2x**（Retina 対応）を推奨。
