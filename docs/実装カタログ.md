## DB テーブル定義

データベース: `data/dev.db` (SQLite)

### training_data

学習用データ (16,512行)。`seed.py` で sklearn から生成。

| カラム | 型 | 説明 |
|---|---|---|
| MedInc | REAL | 世帯収入中央値 |
| HouseAge | REAL | 築年数 |
| AveRooms | REAL | 平均部屋数 |
| AveBedrms | REAL | 平均寝室数 |
| Population | REAL | 地区人口 |
| AveOccup | REAL | 平均居住者数 |
| Latitude | REAL | 緯度 |
| Longitude | REAL | 経度 |
| Price | REAL | 住宅価格 ($100,000 単位) |

### test_data

テスト用データ (4,128行)。カラムは `training_data` と同一。

### model_runs

学習結果の精度記録。`repository.py` で自動作成。

| カラム | 型 | 制約 | 説明 |
|---|---|---|---|
| run_id | TEXT | PRIMARY KEY | Run ID (`YYYYMMDD_HHMMSS_{UUID}`) |
| rmse | REAL | NOT NULL | RMSE |
| r2 | REAL | NOT NULL | R² |
| created_at | TEXT | NOT NULL | 記録日時 (ISO 8601) |

## API エンドポイント

ベース URL: `http://localhost:8000`

### GET /

予測フォーム (Web UI) を返す。

| 項目 | 内容 |
|---|---|
| レスポンス | HTML (Jinja2 テンプレート) |
| 用途 | ブラウザから予測を実行 |

### POST /predict

住宅価格を予測する。

**リクエスト:**

| フィールド | 型 | 説明 |
|---|---|---|
| MedInc | float | 世帯収入中央値 |
| HouseAge | float | 築年数 |
| AveRooms | float | 平均部屋数 |
| AveBedrms | float | 平均寝室数 |
| Population | float | 地区人口 |
| AveOccup | float | 平均居住者数 |
| Latitude | float | 緯度 |
| Longitude | float | 経度 |

**レスポンス:**

```json
{"predicted_price": 4.2103}
```

> 価格の単位は $100,000。4.2103 = 約 $421,030。

**内部処理:** 入力値に対数変換 (`preprocess_input`) → 特徴量生成 (`engineer_features_input`) を適用後、LightGBM で推論。

### GET /health

ヘルスチェック。

```json
{"status": "ok"}
```

## ML モデル

### LightGBM 回帰モデル

| 項目 | 値 |
|---|---|
| アルゴリズム | 勾配ブースティング決定木 (LightGBM) |
| タスク | 回帰 (住宅価格予測) |
| objective | regression |
| metric | rmse |
| learning_rate | 0.05 |
| num_leaves | 31 |
| num_boost_round | 300 |
| 入力特徴量 | 8次元 (前処理後) |
| 出力 | 価格 ($100,000 単位) |
| 保存形式 | `model.lgb` (LightGBM native format) |
| 保存先 | `models/{run_id}/model.lgb` |
| 参照 | `models/latest/model.lgb` (symlink) |

## パイプライン — 各ステップ

`ml/pipeline/main.py` が制御する学習パイプラインの全ステップ:

| # | ステップ | モジュール | 失敗時 |
|---|---|---|---|
| 1 | 設定読み込み | `pipeline/config.py`, `evaluation/config.py` | — |
| 2 | Run ID 生成 | `share/run_id.py` | — |
| 3 | データ取得 | `pipeline/repository.py` | エラーログ → 終了 |
| 4 | 空データチェック | `pipeline/main.py` | エラーログ → 終了 |
| 5 | 前処理 (train) | `pipeline/preprocess.py` | — |
| 6 | 前処理 (test) | `pipeline/preprocess.py` | — |
| 7 | 特徴量エンジニアリング (train) | `pipeline/feature_engineering.py` | — |
| 8 | 特徴量エンジニアリング (test) | `pipeline/feature_engineering.py` | — |
| 9 | W&B 初期化 | `evaluation/tracking.py` | 警告ログ → 続行 |
| 10 | LightGBM 学習 | `trainer/train.py` | エラーログ → 終了 |
| 11 | 評価 (RMSE, R²) | `evaluation/metrics.py` | — |
| 12 | モデル保存 | `trainer/train.py` | — |
| 13 | symlink 更新 | `trainer/train.py` | — |
| 14 | 精度を SQLite に記録 | `pipeline/repository.py` | 警告ログ → 続行 |
| 15 | W&B にログ送信 | `evaluation/tracking.py` | 警告ログ → 続行 |

### 前処理パイプライン (`preprocess.py`)

| # | ステップ | 対象カラム | 処理内容 |
|---|---|---|---|
| 1 | 欠損値補完 | 全特徴量 + Price | 中央値で NaN を埋める |
| 2 | 外れ値キャップ | Population, AveOccup, AveRooms, AveBedrms | 99パーセンタイルで上限カット |
| 3 | 対数変換 | Population, AveOccup | `numpy.log1p()` で右裾を圧縮 |

### 推論時前処理 (`preprocess_input`)

| # | ステップ | 対象カラム | 処理内容 |
|---|---|---|---|
| 1 | 対数変換 | Population, AveOccup | `numpy.log1p()` |

> 推論は単一レコードのため、欠損値補完・外れ値キャップは不要。

### 特徴量エンジニアリング (`feature_engineering.py`)

| # | 生成特徴量 | 計算式 | 意味 |
|---|---|---|---|
| 1 | BedroomRatio | AveBedrms / AveRooms | 寝室が占める割合 |
| 2 | RoomsPerPerson | AveRooms / AveOccup | 1人あたりの部屋数 |

- 学習時: `engineer_features(df)` — DataFrame に2カラムを追加
- 推論時: `engineer_features_input(values)` — dict に2キーを追加

`share/schema.py` の `MODEL_COLS = FEATURE_COLS + ENGINEERED_COLS` (10特徴量) でモデルに渡す。

## Evaluation — 評価定義

### 評価指標

numpy 自前実装 (scikit-learn 不使用)。`ml/evaluation/metrics.py` で定義。

| 指標 | 関数 | 計算式 | 理想値 |
|---|---|---|---|
| RMSE | `rmse(y_true, y_pred)` | `sqrt(mean((y_true - y_pred)²))` | 0 に近い |
| R² | `r2_score(y_true, y_pred)` | `1 - SS_res / SS_tot` | 1.0 に近い |

### R² のエッジケース

| 条件 | 戻り値 |
|---|---|
| SS_tot = 0 かつ SS_res = 0 (完全一致 + 定数) | `0.0` |
| SS_tot = 0 かつ SS_res > 0 (定数だが誤差あり) | `NaN` |

### 集約関数

| 関数 | 戻り値 |
|---|---|
| `evaluate(y_true, y_pred)` | `{"rmse": float, "r2": float}` (小数点4桁で丸め) |
| `save_metrics(metrics, path)` | JSON ファイルに保存 |

### 記録先

| 記録先 | 内容 | 必須 |
|---|---|---|
| `models/{run_id}/metrics.json` | RMSE, R², run_id | 常に保存 |
| `data/dev.db` → `model_runs` テーブル | run_id, RMSE, R², created_at | 常に保存 |
| W&B | RMSE, R², run_id | オプション (API キー依存) |
